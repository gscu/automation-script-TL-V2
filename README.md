# GONet TELUS Bandwidth Report Automation

Automates the twice-daily bandwidth reports for the GUELPH and KINGSTON
data-center interfaces on the TELUS eHealth portal:

1. Opens the eHealth **Standard Reports** page in an automated browser
   (Playwright) and generates a 24-hour trend report for each interface.
2. Downloads each report PDF (with retries, since the portal is often
   slow to have them ready).
3. Parses total bandwidth plus max/min Bits In/Out from each PDF and
   flags anything that **peaked ≥ 70%** of capacity or **dipped below
   1 Mbps**.
4. Opens a pre-addressed Outlook email from an `.oft` template with the
   PDFs attached and the color-coded alert summary at the top — you just
   review and hit Send.

A desktop app (**Bandwidth Report Manager**) wraps all of this with
one-click run buttons, live console output, scheduling controls, and a
settings screen.

> **New machine? Non-technical user?** Follow **[EASY_SETUP.md](EASY_SETUP.md)** —
> it's the plain-English version of everything below.

## Quick start

```
1. Install Python 3 (tick "Add Python to PATH")
2. Put this folder somewhere user-writable (Documents, Desktop — NOT Program Files)
   and make sure the two .oft Outlook templates are in it
3. Double-click setup.bat  →  answer the questions
4. Open "Bandwidth Report Manager" from the Desktop shortcut
```

`setup.bat` installs the Python packages and the Playwright Chromium
engine, adds a Desktop shortcut, then hands off to `Setup_script.py`
which asks for your reports folder, eHealth credentials, and whether to
schedule the reports in Windows Task Scheduler (Morning 10:58, Afternoon
14:58 by default).

## Day-to-day use

- **Manually:** open the app → **Run Morning/Afternoon Report Now** →
  watch the live Console at the bottom → review the Outlook email it opens.
- **Automatically:** if you enabled scheduling during setup, the reports
  run daily on their own; use the app's Task Scheduler panel to check
  status or flip them on/off.
- **Change settings:** app sidebar → **⚙ Options** (credentials and
  reports folder). Changes are written straight into the two report
  scripts — no separate config file.

## Configuration knobs (edit the report scripts)

All in `Morning BW Reports.py` and `Afternoon BW Reports.py` — keep the
two files in sync:

- `INTERFACES` — the list of interfaces to report on. Add/remove/replace
  lines as the data centers change.
- `EXCLUDE_DIP` — interfaces that are known to idle low and shouldn't
  trigger "dipped below 1 Mbps" alerts.
- `threshold_pct` (in `run()`) — the peak-alert threshold, default 70%.
- `MIN_THRESHOLD_GBPS` (in `run()`) — the dip-alert floor, default 0.001 (1 Mbps).
- `FORM_PAGE` — the portal URL; update it here if TELUS changes it.
- The special-case override in `parse_bandwidth_from_pdf()` pins the
  `...SGWP...` interface to 10 Gbps because its PDF doesn't state total BW.

Schedule times live in `SCHEDULED_TASKS` in `Setup_script.py`
(re-run setup after changing them).

## Good to know

- **The password is stored encrypted** (Windows DPAPI) inside the two
  report scripts as an `ENC:...` value. The encryption is tied to your
  Windows user on that PC — scheduled tasks decrypt it fine, but a copy
  of the folder on another machine or user account can't; just re-enter
  the password there via setup or ⚙ Options. The username is stored in
  plain text, and a hand-typed plain-text password still works too.
  Both setup and ⚙ Options have an **"Encrypt password"** toggle — leave
  it on unless you're deliberately pre-configuring a folder that will be
  moved to a different machine (re-encrypt it there afterwards).
  Still keep configured copies out of shared drives and public repos.
- The report scripts need the machine's own Python plus Outlook desktop;
  they can't be frozen into the exe (see `INSTALLER_GUIDE.md`).
- The portal browser opens **visible** (`headless=False` in `run()`).
  That's intentional — the portal has been flaky headless — but you can
  flip it to `True` to run in the background.
- The email step **displays** the message (`mail.Display()`) rather than
  auto-sending, so a human always gets a final look.

## Troubleshooting

Most failures show up in the app's **Console** (bottom of the window) in red.

| Symptom | Likely cause |
|---|---|
| Fails almost immediately at the portal page | Wrong eHealth username/password (⚙ Options), or portal URL changed |
| "report not ready (attempt N/10)" over and over | Portal is slow/down — try again later; it retries 10× per interface |
| "BW not found in …" warning | The PDF for that interface doesn't state total bandwidth — parsed values still save, percentages read 0 |
| Report finishes but no email appears | Outlook desktop not installed, or the `.oft` template files are missing from this folder |
| Scheduled task shows "Not found" in the app | Scheduling was skipped during setup — re-run `Setup_script.py` and answer `y` |
| "saved eHealth password could not be decrypted" | The folder was copied from another PC/user — re-enter the password in ⚙ Options or re-run setup |
